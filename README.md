# QA-Assistant (Kaspersky LLM Edition)

**QA-Assistant** - веб-приложение для проработки тест-дизайнов, интегрированное с корпоративным ИИ-сервисом Kaspersky.

По вопросам и рекомендациям писать: eduard.avagimov@kaspersky.com

---

## Стек технологий

*   **Backend**: Python 3.12, FastAPI, Pydantic, Pandas, OpenAI SDK (интегрировано с Kaspersky LLM).
*   **Frontend**: React (Vite), TypeScript, Vanilla CSS (Premium Glassmorphism), Lucide Icons, Axios.
*   **Контейнеризация**: Docker, Docker Compose.

## Пошаговая инструкция по запуску с нуля

### Шаг 1: Клонирование репозитория
Склонируйте репозиторий проекта на ваш компьютер:
```bash
git clone https://github.com/Ava-Edvvard/qa-assistant-avp-ru.git
cd qa-assistant-avp-ru
```

### Шаг 2: Настройка конфигурационного файла .env
Перейдите в директорию backend и настройте файл конфигурации:
1. Создайте файл `.env` (если его нет) или отредактируйте существующий `backend/.env`.
2. Настройте параметры подключения к Kaspersky LLM:

```
# Используем кастомный OpenAI-совместимый провайдер
LLM_PROVIDER=custom

# URL API Kaspersky LLM
CUSTOM_BASE_URL=https://llm.kaspersky-labs.com/v1/

# API-ключ (Инструкция по получению: https://confluence.kaspersky.com/spaces/LS/pages/1577822276/%F0%9F%A4%96%D0%A0%D0%B0%D0%B1%D0%BE%D1%82%D0%B0+%D1%87%D0%B5%D1%80%D0%B5%D0%B7+API)
CUSTOM_API_KEY=ваш-реальный-ключ-api

# Модель по умолчанию
CUSTOM_MODEL=llama-3.3-70B-instruct
```

> **Офлайн-режим (Mock)**: Если вы не укажете реальные API-ключи (оставите значение по умолчанию `mock-key-replace-with-your-real-key`), система автоматически перейдет на умную локальную заглушку. Она полностью имитирует работу ИИ, включая корректное объединение требований и генерацию тест-кейсов.

---

### Шаг 3: Запуск приложения

Выберите один из трех удобных вариантов запуска:

#### Вариант А: Быстрый запуск через исполняемые файлы (Рекомендуется)
Этот способ автоматически установит библиотеки и запустит бэкенд и фронтенд локально.

*   **На Windows:**
    Просто дважды кликните по файлу `start.bat` в корневой папке проекта.
*   **На Linux/macOS:**
    Дайте скрипту права на выполнение и запустите его в терминале:
    ```bash
    chmod +x start.sh
    ./start.sh
    ```

> **Как это работает**: Скрипты автоматически проверят наличие установленных Python и Node.js. При их отсутствии скрипт предупредит и даст ссылки на установку. Если всё установлено, скрипт настроит виртуальное окружение `venv`, установит зависимости, запустит серверы и сам откроет браузер на странице приложения.

---

#### Вариант Б: Запуск через Docker (Без установки Python и Node.js)

Этот способ изолирует приложение в контейнерах. Установка Python, Node.js или каких-либо библиотек на ваш физический компьютер не требуется.

##### 1. Требования (что нужно установить)
* Установите **Docker Desktop** для вашей операционной системы: [Скачать Docker Desktop](https://www.docker.com/products/docker-desktop/).
* **Для пользователей Windows:**
  * Убедитесь, что в BIOS/UEFI вашего ПК включена аппаратная виртуализация (Intel VT-x или AMD-V).
  * Убедитесь, что в системе установлен WSL 2 (Windows Subsystem for Linux).
  * Если при запуске Docker Desktop сообщает, что версия WSL устарела, откройте **Командную строку от имени администратора** и выполните команду для принудительного обновления:
    ```cmd
    wsl --update --web-download
    ```
    После этого перезапустите Docker Desktop.

##### 2. Запуск платформы
1. Запустите Docker Desktop и убедитесь, что его движок запущен (зеленый индикатор в углу).
2. Выполните сборку и запуск контейнеров в фоновом режиме из корневой папки проекта:
   ```bash
   docker compose up -d --build
   ```
3. Откройте приложение в браузере: [http://localhost:5173](http://localhost:5173)

##### 3. Полезные команды для управления контейнерами
* **Просмотр состояния запущенных сервисов:**
  ```bash
  docker compose ps
  ```
* **Просмотр логов работы бэкенда и фронтенда:**
  ```bash
  docker compose logs -f
  ```
* **Остановка работы (освобождение портов и памяти):**
  ```bash
  docker compose down
  ```
* **Остановка с полной очисткой собранных образов (для экономии места):**
  ```bash
  docker compose down --rmi local
  ```
* **Очистка зависших сборок и кэша сборщика BuildKit (освобождает место на диске):**
  ```bash
  docker builder prune -f
  ```

#### Вариант В: Ручной запуск (для разработчиков)

1.  **Запуск Backend (FastAPI)**:
    ```bash
    cd backend
    python -m venv venv
    
    # Активация venv:
    # Windows:
    .\venv\Scripts\activate
    # Linux/macOS:
    source venv/bin/activate
    
    pip install -r requirements.txt
    uvicorn app.main:app --reload --port 8000
    ```
    API-документация будет доступна на http://127.0.0.1:8000/docs

2.  **Запуск Frontend (React)**:
    ```bash
    cd frontend
    npm install
    npm run dev
    ```
    Интерфейс откроется на http://127.0.0.1:5173

## Функционал платформы

1.  **Новый тест-дизайн** (5 шагов):
    *   **Шаг 1**: Загрузка исходных требований (текст, Excel-таблицы или скриншоты интерфейсов с использованием моделей компьютерного зрения).
    *   **Шаг 2**: Формирование интерактивной матрицы требований. Здесь можно редактировать, удалять или добавлять требования вручную.
    *   **Шаг 3**: Уточняющие вопросы от ИИ. Система анализирует требования на наличие логических дыр и задает вопросы (с возможностью исключить вопрос кнопкой "Не рассматривается").
    *   **Шаг 4**: Генерация структурированных тест-сценариев (Шаги, Предусловия, Ожидаемый результат, Покрытие требований).
    *   **Шаг 5 (Результаты)**: Интерактивный просмотр сгенерированной матрицы трассируемости (с экспортом в .CSV для Excel), просмотр тест-сценариев в виде наглядного аккордеона с интерактивными тултипами по требованиям и копированием всего тест-плана в буфер обмена в один клик.
2.  **Актуализация существующего тест-дизайна** (6 шагов):
    *   Включает все шаги создания тест-дизайна с дополнительным **Этапом 5 (Сводка изменений / Diff Report)**: подробное сравнение старого тест-плана с новым, визуальный вывод статистики изменений по приоритетам (П1, П2, П3) и возможность быстрого копирования отчета об изменениях в один клик.
