# Спецификация API - QA-Assistant

Бэкенд предоставляет несколько эндпоинтов для анализа требований, генерации вопросов, создания тест-кейсов и их сравнения.

---

## 1. Анализ требований и файлов
Извлекает структурированный список требований из сырого текста и вложений.

- **Эндпоинт**: `POST /api/design/parse-requirements`
- **Тело запроса (Multipart Form-Data)**:
  - `requirements_text`: string (обязательное)
  - `additional_info`: string (опциональное)
  - `files`: бинарные файлы (Excel, изображения) (опциональное)
  - `llm_provider`: string (опциональное, например: "openai", "gemini", "custom")
  - `llm_api_key`: string (опциональное)
  - `llm_base_url`: string (опциональное)
  - `llm_model`: string (опциональное)
- **Ответ** (`200 OK`):
  ```json
  {
    "requirements": [
      {
        "id": "RQ-01",
        "description": "Тестовое требование 1",
        "cases_count": 0
      },
      {
        "id": "RQ-02",
        "description": "Тестовое требование 2",
        "cases_count": 0
      }
    ]
  }
  ```

---

## 2. Генерация уточняющих вопросов
Опрашивает LLM для выявления неопределенностей, серых зон и логических пропусков в матрице требований.

- **Эндпоинт**: `POST /api/llm/generate-questions`
- **Тело запроса**:
  ```json
  {
    "requirements": [
      {
        "id": "RQ-01",
        "description": "Тестовое требование 1"
      }
    ],
    "llm_config": {
      "provider": "gemini",
      "api_key": "AIzaSy...",
      "base_url": null,
      "model": "gemini-2.5-flash"
    }
  }
  ```
  *(Примечание: блок `llm_config` является необязательным)*
- **Ответ** (`200 OK`):
  ```json
  {
    "questions": [
      {
        "id": "Q-01",
        "requirement_id": "RQ-01",
        "question": "Каково ожидаемое поведение при вводе неверного пароля?"
      }
    ]
  }
  ```

---

## 3. Генерация тест-сценариев
Создает структурированные тест-кейсы на основе требований и ответов на уточняющие вопросы.

- **Эндпоинт**: `POST /api/llm/generate-scenarios`
- **Тело запроса**:
  ```json
  {
    "requirements": [
      {
        "id": "RQ-01",
        "description": "Тестовое требование 1"
      }
    ],
    "answers": [
      {
        "question_id": "Q-01",
        "question": "Каково ожидаемое поведение при вводе неверного пароля?",
        "answer": "Отобразить ошибку 'Неверный логин или пароль'"
      }
    ],
    "llm_config": {
      "provider": "gemini",
      "api_key": "AIzaSy...",
      "base_url": null,
      "model": "gemini-2.5-flash"
    }
  }
  ```
  *(Примечание: блок `llm_config` является необязательным)*
- **Ответ** (`200 OK`):
  ```json
  {
    "scenarios": [
      {
        "id": "TC-001",
        "name": "Открытие страницы авторизации",
        "priority": "П1",
        "preconditions": [
          "Пользователь находится на главной странице"
        ],
        "steps": [
          "Нажать на кнопку Войти"
        ],
        "expected_results": [
          "Страница авторизации успешно открылась"
        ],
        "coverage": ["RQ-01"]
      }
    ]
  }
  ```

---

## 4. Сравнение сценариев
Сопоставляет старые тест-кейсы с новыми сгенерированными сценариями и формирует отчет об изменениях.

- **Эндпоинт**: `POST /api/llm/compare-scenarios`
- **Тело запроса**:
  ```json
  {
    "old_scenarios_text": "TC-001: Тестовый сценарий...",
    "new_scenarios": [
      {
        "id": "TC-001",
        "name": "Открытие страницы авторизации",
        "priority": "П1",
        "preconditions": ["..."],
        "steps": ["..."],
        "expected_results": ["..."],
        "coverage": ["RQ-01"]
      }
    ],
    "llm_config": {
      "provider": "gemini",
      "api_key": "AIzaSy...",
      "base_url": null,
      "model": "gemini-2.5-flash"
    }
  }
  ```
  *(Примечание: блок `llm_config` является необязательным)*
- **Ответ** (`200 OK`):
  ```json
  {
    "changes_summary": "Сводка изменений:\n1. TC-001: Изменены предусловия...\n2. Добавлен новый тест-кейс TC-002...",
    "added": ["TC-002"],
    "removed": [],
    "modified": ["TC-001"]
  }
  ```

---

## 5. Получение настроек LLM сервера
Возвращает конфигурацию нейросети, настроенную по умолчанию на сервере.

- **Эндпоинт**: `GET /api/llm/info`
- **Тело запроса**: отсутствует
- **Ответ** (`200 OK`):
  ```json
  {
    "provider": "gemini",
    "model": "gemini-2.5-flash",
    "is_mock": false
  }
  ```

---

## 6. Получение списка доступных моделей
Опрашивает динамического ИИ-провайдера для возвращения списка доступных ему моделей.

- **Эндпоинт**: `POST /api/llm/models`
- **Тело запроса**:
  ```json
  {
    "provider": "gemini",
    "api_key": "AIzaSy...",
    "base_url": null
  }
  ```
- **Ответ** (`200 OK`):
  ```json
  {
    "models": [
      "gemini-2.5-flash",
      "gemini-2.5-pro"
    ]
  }
  ```
